📤 Guide Upload Assets Compilés

Solution complète pour corriger l'installateur AdminLicence sur cPanel

🎯 Vous avez identifié le vrai problème !

Effectivement, compiler sur cPanel ne fonctionne pas bien. La meilleure pratique est de :

  1. ✅ Compiler en local (déjà fait)
  2. 📤 Uploader les assets compilés vers cPanel
  3. 🔧 Corriger le fichier .env sur le serveur

📁 Étape 1 : Fichiers à uploader

Vous devez uploader ces répertoires/fichiers vers votre cPanel :

📤 Étape 2 : Méthodes d'upload

🔹 Méthode 1 : Gestionnaire de fichiers cPanel

  1. Connectez-vous à votre cPanel
  2. Ouvrez le Gestionnaire de fichiers
  3. Naviguez vers public_html/
  4. Créez le dossier build s'il n'existe pas
  5. Uploadez manifest.json dans public_html/build/
  6. Créez le dossier assets dans public_html/build/
  7. Uploadez tous les fichiers du dossier assets

🔹 Méthode 2 : FTP/SFTP (plus rapide)

  1. Utilisez FileZilla ou WinSCP
  2. Connectez-vous à votre serveur
  3. Naviguez vers /public_html/
  4. Glissez-déposez le dossier build complet

🔹 Méthode 3 : Compression ZIP

  1. Compressez le dossier build en ZIP
  2. Uploadez le ZIP via cPanel
  3. Extrayez-le dans public_html/

Après l'upload des assets, corrigez le fichier .env :

🔹 Option A : Via Gestionnaire de fichiers cPanel

  1. Ouvrez le fichier .env à la racine
  2. Modifiez les permissions à 666 ou 777
  3. Éditez le contenu si nécessaire

✅ Étape 4 : Vérification

🔍 Checklist de vérification :

  • Le dossier public_html/build/ existe
  • Le fichier public_html/build/manifest.json existe
  • Le dossier public_html/build/assets/ contient tous les fichiers
  • Le fichier .env est accessible en écriture
  • L'URL https://adminlicence.eu/build/manifest.json est accessible

🧪 Tests à effectuer :

  1. Test assets : https://adminlicence.eu/build/manifest.json
  2. Test installateur : https://adminlicence.eu/install/install_new.php
  3. Vérifier étape 2 : Plus d'erreur "❌ .env (Écriture) Erreur"
  4. Vérifier étape 5 : Affichage correct des informations

🎉 Résultat attendu

Après ces étapes :

  • ✅ Étape 2 : "✅ .env (Écriture) OK"
  • ✅ Étape 5 : Affichage correct des informations de base de données et administrateur
  • ✅ Installation finale : Fonctionne sans erreur "HTML détecté"

💡 Pourquoi cette solution fonctionne :

  • Assets compilés : Tous les CSS/JS sont optimisés pour la production
  • Pas de Node.js requis : Le serveur cPanel n'a besoin que des fichiers statiques
  • Performance optimale : Les assets sont minifiés et compressés
  • Compatibilité cPanel : Aucune dépendance système requise

⚠️ Important :

Chaque fois que vous modifiez les fichiers CSS/JS en local, vous devrez :

  1. Exécuter npm run build en local
  2. Re-uploader le dossier build vers cPanel
